Section 1: My Work As A Student

I began studying for a career in Computer Application Development in 2014. Towards this goal, I used the online-education service Udacity almost exclusively.

At first, I used their free courseware, and I really got some good practice doing that. However, when they introduced "nanodegrees" in October 2014 I elected to purse the Front-End Web Development Nanodegree. This was a new type of credential program aimed at giving students the skills to enter Front-End Web Development careers and jobs. You can read more about Udacity's nanodegrees here, and about the "FEND" program here.

Section 1a will be showcasing the projects and code I wrote as a nanodegree student, and Section 1b will list and explain the other Udacity courses I have completed.

Unit 4: Optimizing Content for Fast User Experience

Overview of Courses and What I Learned

  • Udacity Course: Web Performance Optimization

    This course was taught by Ilya Grigorik. He really impressed me. He now works for Google as a Developer Advocate, having built up a lot of experience with the network technology behind the internet. He even wrote an O'Reilly book on High-Performance Browser Networking. His blog archives are always great for web developers to read when they have time to go deep on technical issues, and his Twitter feed is always tells you what he considers to be the latest good information on such issues.

    This course showed us an excellent learning portal for web developers built by Google. It has many hours of content to learn from, surveying very many best practices for developers.

    Thanks to this course, I am now also fluent with the vendor documentation for "Dev Tools" utilities both in Chrome and in Firefox.

    Finally, this course also introduced us to the PageSpeed Insights tool, which runs a battery of automated test on any page you are developing to help you make sure you aren't inadvertently and needlessly creating networking bottlenecks for delivering pages to users. This is especially of concern for mobile-friendly development.

  • Browser Rendering Optimization

Final Project

GitHub repository for project code:
https://github.com/prather-mcs/nano/tree/master/project-4

The final project for this course required us to make important changes to a multi-page site to bring its PageSpeed Insights score (PSI is described above) from 30 to almost 100, while still perfectly preserving the original appearance and functionality of the site.

To accomplish this, I had make optimizations to (a) delivery of webfonts, (b) delivery of images, (c) delivery and sync/async behavior of JavaScript files, and (d) delivery of CSS style rules. Finally, I also re-wrote the JavaScript animation-control routines on the "pizzeria" page, to change it from being extremely janky to perfectly fluid at 60 fps animation rate. Animation there occurred while you scroll the page and stops when you stop scrolling. The weaker your computer's graphics and CPU resources, the greater the difference you will notice due to my changes.

  • See the original, un-optimized version:
    https://prather-mcs.github.io/udacity-units/unit-4/final-before
  • See the same page after it has been fully optimized:
    https://prather-mcs.github.io/udacity-units/unit-4/final-after
